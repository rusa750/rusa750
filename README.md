## What I work on

I like building systems that solve operational problems.

That usually means taking a process that is slow, messy, manual, or hard to track, and turning it into something that runs in a more structured way.

Things I usually work with:

- Business automation
- Trading execution systems
- Webhook-based workflows
- Broker and exchange APIs
- Internal dashboards
- Reporting systems
- Google Sheets automation
- Backend tools
- Data flow cleanup

---

## Problems I care about

A lot of businesses lose time because the same things are done manually every day.

Someone copies data from one place to another.  
Someone checks if something happened.  
Someone sends the same update again.  
Someone has to open five tools to understand what is going on.

That is usually where automation makes sense.

The goal is simple:

```txt
Less manual work
Faster execution
Cleaner data
Fewer mistakes
Better visibility
More control
```

## Systems I have worked on

### Trading automation

I have worked on systems that connect TradingView alerts to broker or exchange execution.

The important part is making the flow safer, faster, and easier to understand.

Improvements I focus on:

- Faster signal handling
- Duplicate signal protection
- Safer order logic
- Clear execution logs
- Demo/live separation
- Webhook validation
- Better error handling
- Less manual checking

A trading system should make it clear what happened, why it happened, and what state it is in now.

---

### Business automation

I build workflows that reduce repetitive work inside a business.

Examples of improvements:

- Moving data automatically between tools
- Cleaning up messy workflow logic
- Reducing manual admin
- Making reporting easier
- Creating better handoff between steps
- Making failed steps easier to find and fix

I care a lot about keeping systems simple.

If a workflow becomes too heavy, too slow, or too hard to debug, it becomes a problem by itself.

---

### Internal tools and dashboards

I also build simple internal tools for businesses that need better control over their operations.

That can include:

- Dashboards
- Forms
- Admin panels
- Reporting pages
- API-connected tools
- Lightweight business systems

The point is to give the business a place where the important actions and information are easy to access.

---

## How I build

```txt
Understand the process
Find the repeated steps
Remove what is unnecessary
Automate the useful parts
Add checks and logging
Make it easy to debug
Keep improving the system
```

I prefer practical systems that are easy to use, easy to understand, and strong enough to run under real conditions.
